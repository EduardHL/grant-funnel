You are a structured data extraction assistant. Your task is to extract a list of grantees (organizations that receive funding) from a grant-maker's website.

## Input

The following URL is the portfolio/grantees page of a grant-making organization:
{{PORTFOLIO_URL}}

## Instructions

1. **Identify the grant-maker**: Extract the grant-making organization's name, top-level website URL, and a brief description of their mission or focus (1–2 sentences, taken from their site).

2. **Extract all grantees**: For each grantee listed on the page, extract:
   - **name**: The grantee organization's name, exactly as displayed.
   - **website**: The grantee's own website URL (not a link back to the grant-maker's site).
   - **focusArea**: The primary focus area of the grantee's work. Classify into exactly one of the following values: "Health", "Nutrition", "Education", "Livelihoods", "WASH", "Environment", or "Other". Choose the category that best represents the organization's core mission or the specific intervention being funded. Use "Health" for medical care, disease prevention, and health systems. Use "Nutrition" for food supplementation, food fortification, malnutrition treatment, and dietary programs. Use "Education" for schooling, training, and learning initiatives. Use "Livelihoods" for farmer assistance, income generation, economic empowerment, and poverty alleviation. Use "WASH" for water, sanitation, and hygiene programs. Use "Environment" for conservation, climate, and environmental protection. Use "Other" only when the work clearly does not fit any of the above categories. If multiple areas apply, select the one most central to the grantee's intervention.
   - **description**: A concise summary of what the grantee does, based on text from the page.
   - **geographies**: A list of countries where the grantee implements their intervention. If the organization operates globally or in a broad region without listing specific countries, return the region or descriptor (e.g., "Sub-Saharan Africa", "Global"). Return an empty list if no geographic information is provided. Do not infer countries from the organization's headquarters alone—focus on where programs are actively delivered.

3. **Enrich from grantee websites**: After extracting the initial list from the grant-maker's portfolio page, visit each grantee's own website (if available) to fill in or improve any incomplete fields — especially `focusArea`, `description`, and `geographies`. The grant-maker's page often provides only a brief mention; the grantee's own site will typically have more detailed and accurate information about their mission, programs, and countries of operation. Prefer information from the grantee's own site when it conflicts with or is more specific than what the grant-maker provides.

4. **Be thorough**: Include every grantee listed on the page. If the page uses lazy loading, pagination, or tabs to show additional grantees, note this in the `extraction_notes` field.

5. **Do not fabricate data**: Do not guess URLs or invent descriptions. Use `null` for any field you cannot determine, or an empty list `[]` for geographies.

## Output

Save the result as a local JSON file named using the grant-maker's name in kebab-case (e.g., `mulago-foundation.json`). The file should contain **only** a valid JSON object with the following structure — no markdown fencing, no commentary outside the JSON:

{
  "grant_maker": {
    "name": "string",
    "website": "string (top-level domain, e.g. https://example.org)",
    "description": "string"
  },
  "grantees": [
    {
      "name": "string",
      "website": "string | null",
      "focusArea": "Health | Nutrition | Education | Livelihoods | WASH | Environment | Other",      
      "description": "string | null",
      "geographies": ["string"] 
    }
  ],
  "extraction_notes": "string | null (flag any issues: pagination, incomplete loading, ambiguous entries, etc.)"
}
